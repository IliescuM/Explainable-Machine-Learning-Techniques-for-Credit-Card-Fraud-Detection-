\section{Goals and Epistemological Framing}
Whereas methodological substance was formalised algebraically in Chapter~3, this chapter investigates how empirical quantities instantiate those abstractions without elevating ancillary software engineering to coequal scholarly status \cite{fraud_handbook,xai_survey}. The aims are therefore threefold:
\begin{enumerate}
    \item demonstrate reproducible pipelines mapping anonymised transactional predictors $\bigl(\mathrm{time}, \{v_{\ell}\}_{\ell=1}^{28},\mathrm{amount}\bigr)$ to probabilistic forecasts $\hat{y}$ under extreme imbalance;
    \item relate scalar evaluation summaries (ROC, PR, $F_1$, threshold sensitivities) to the asymmetric cost motifs discussed theoretically;
    \item interpret explanation vectors $e(x)$ critically, highlighting limitations inherited from perturbation neighbourhoods or correlation proxies.
\end{enumerate}
Observed numerical outcomes should be interpreted as \emph{cases} illuminating theoretical claims rather than exhaustive industrial certification.

\section{Dataset Schema and Confidentiality Constraints}
Public fraud corpora routinely publish anonymised PCA coordinates instead of semantic entity descriptors \cite{credit_card_dataset,fraud_handbook}; consequently variables $v_\ell$ lack direct lexical mapping to merchants or cardholders, yet statistical dependence structures remain scientifically informative.

Experiments emulate canonical ULB preprocessing: raw CSV rows represent independent transactions labelled by latent fraud adjudication protocols. Temporal stratification assumptions should be audited before claiming deployment readiness; leakage across sliding windows materially inflates retrospective metrics absent careful bookkeeping.

\section{Experimental Setup and Evaluation Protocol}
To ensure that model performance claims are reproducible and academically defensible, the evaluation process follows a fixed protocol. The dataset is partitioned into training, validation, and test subsets using temporal constraints where possible, in order to avoid information leakage from future transactions into historical training data. All preprocessing steps applied during training are serialized and reused during inference.

Given the imbalanced nature of fraud detection, model quality is assessed using metrics that reflect both detection power and operational impact. In addition to precision and recall, the evaluation includes F1-score and precision-recall area under curve (PR-AUC), because these measures are more informative than accuracy in highly skewed class distributions \cite{fraud_handbook}. Threshold analysis is performed to identify the operating point that balances fraud capture with analyst workload.

The baseline comparison includes at least one simple model (for example, logistic regression) and one strong non-linear model (for example, gradient boosting). This comparative design supports a credible argument that final model selection is driven by evidence rather than by implementation convenience.

\section{Evaluation Metrics}
Let $TP$, $FP$, $TN$, and $FN$ denote true positives, false positives, true negatives, and false negatives, respectively. Canonical definitions follow:
\begin{equation}
\text{Precision} = \frac{TP}{TP + FP},\qquad \text{Recall} = \frac{TP}{TP + FN},\qquad
F_1 = 2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}} .
\end{equation}
ROC-AUC measures ranking quality across negatives and positives irrespective of prevalence; denote it $\mathrm{AUC}_{\mathrm{ROC}}$.
\begin{equation}
\mathrm{AUC}_{\mathrm{ROC}} = \int_0^1 \text{TPR}(\alpha)\, \mathrm{d}(\text{FPR}(\alpha)) .
\end{equation}
Average precision aggregates precision--recall operating points analogous to discrete summation formulae articulated in Chapter~2. Together they stress complementary geometry: ROC emphasises discriminative ordering whereas PR reacts sharply to imbalance-induced false-positive inflation \cite{fraud_handbook}.

Precision encodes investigative purity; recall expresses fraud containment. Institutional utility demands joint satisfaction under budgeted alert volume; thus threshold sweeps annotate metric surfaces referenced below.

\section{Results and Discussion}
This section interprets empirical outcomes relative to methodological objectives linking predictive separation, asymmetric costs, and defensible explanatory narratives. The quantitative narrative should be completed only with measured values obtained from the final experiments.

\subsection{Predictive Performance Interpretation}
Using Table~\ref{tab:model-comparison}, the first discussion point should compare the baseline model against the selected final model. If the final model improves recall while maintaining acceptable precision, this indicates better fraud capture without an unmanageable increase in false alerts. If precision improves at similar recall levels, the model is likely reducing analyst workload while preserving detection coverage.

The dissertation should explicitly report absolute differences between models. For example:
\begin{itemize}
    \item Precision improvement: $[P_{\text{final}} - P_{\text{baseline}}]$.
    \item Recall improvement: $[R_{\text{final}} - R_{\text{baseline}}]$.
    \item F1-score improvement: $[F1_{\text{final}} - F1_{\text{baseline}}]$.
\end{itemize}
These comparisons provide evidence that model selection is performance-driven rather than arbitrary.

\subsection{Operational Threshold Analysis}
Beyond model choice, threshold selection has direct operational impact. The final thesis text should justify the selected threshold $\tau^\star$ by linking it to expected analyst capacity and fraud-risk tolerance. A practical interpretation should answer the following:
\begin{itemize}
    \item How many alerts per day are generated at $\tau^\star$?
    \item What percentage of known fraud cases is captured at $\tau^\star$?
    \item How does changing $\tau$ affect false-positive volume?
\end{itemize}
This analysis demonstrates that the solution is not only statistically valid but operationally deployable.

\subsection{Business and Compliance Perspective}
Results should also be interpreted from a governance perspective. The combination of prediction logs, model versioning, and explanation metadata provides a traceable decision trail that supports audit readiness. The final chapter text should therefore include a short statement on how these artifacts satisfy internal review and accountability requirements in financial environments.

\subsection{Limitations of Reported Results}
The discussion must acknowledge that results are conditional on the available dataset, labeling quality, and observation period. If the dataset is geographically or temporally narrow, external validity is limited. If fraud patterns shift, periodic retraining and threshold recalibration become mandatory. Presenting these limitations strengthens academic credibility.

\section{Threats to Validity}
Three validity tensions persist. Temporal covariate shift erodes calibrated probabilities absent monitoring. Explanation artefacts summarise model sensitivity, yet remain non-causal and potentially unstable across neighbours in $\mathcal{X}$ \cite{xai_survey}. Human factors---investigator trust curves, escalation protocol elasticity---moderate uptake independent of ROC geometry.

\section{Minimal Reproducibility Artefacts}
A companion source repository provides executable trainers, serialised artefacts, REST scoring surfaces, and interactive visualisations purely to mirror the methodological pipeline; exhaustive source listings are orthogonal to dissertation arguments and remain external reading. Quantitative filings in Table~\ref{tab:model-comparison} should therefore be regenerated from scripted seeds rather than hand-transcribed.
